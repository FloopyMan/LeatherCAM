# План разработки CAM-приложения для CNC 3018 (клише для тиснения кожи)

## 1. Обзор проекта

**Название (рабочее):** LeatherCAM

**Цель:** Кроссплатформенное (Linux + Windows) десктоп-приложение, которое из растрового или векторного изображения генерирует G-code для станка CNC 3018 (контроллер GRBL) для изготовления клише для горячего/холодного тиснения кожи.

**Входные данные:**
- Изображение (PNG, JPG, BMP) или вектор (SVG, DXF).
- Параметры материала (тип, толщина).
- Параметры фрезы (тип, диаметр, угол, длина режущей части).
- Параметры фрезеровки (глубина, шаг, скорость подачи, скорость шпинделя).

**Выходные данные:**
- Файл `.gcode`, совместимый с GRBL 1.1 (CNC 3018).
- Предпросмотр траектории (2D + псевдо-3D).
- Оценка времени работы.

---

## 2. Целевая платформа и стек

### 2.1 Выбор языка и фреймворка

Рекомендуемый вариант — **Python 3.11+ + PySide6 (Qt 6)**:
- Кроссплатформенность из коробки (Linux, Windows, macOS).
- Богатая графика (QGraphicsView для предпросмотра).
- Огромная экосистема для обработки изображений и векторов.
- Простая упаковка через PyInstaller.

Альтернативы (если будут проблемы с производительностью):
- **C++ + Qt 6** — быстрее, но дольше разработка.
- **Electron + TypeScript** — удобный UI, но большой бинарник и медленная обработка пикселей.
- **Rust + egui/Tauri** — производительно, но меньше готовых библиотек для CAM.

### 2.2 Ключевые зависимости (Python)

| Назначение | Библиотека |
|---|---|
| GUI | PySide6 |
| Изображения | Pillow, OpenCV |
| Численные операции | NumPy, SciPy |
| Вектор (SVG) | svgpathtools, svgelements |
| DXF | ezdxf |
| Геометрия и оффсеты | Shapely, pyclipper |
| Триангуляция/обход | scikit-image (skeletonize, contour) |
| 3D-предпросмотр | PyOpenGL или Qt3D |
| Отправка на станок (опционально) | pyserial |
| Тесты | pytest |
| Упаковка | PyInstaller, Inno Setup (Windows), AppImage (Linux) |

## 4. Функциональность по этапам

### Этап 1. MVP (минимально жизнеспособный продукт)

**Цель:** загрузить чёрно-белый PNG → выбрать фрезу и глубину → получить G-code.

**Задачи:**
1. Создать репозиторий, настроить виртуальное окружение, CI (GitHub Actions: lint + tests на ubuntu и windows).
2. Каркас приложения PySide6: главное окно, меню, центральный QGraphicsView.
3. Загрузка PNG/JPG, масштабирование под физический размер (мм/дюймы, DPI).
4. Бинаризация изображения с порогом, инверсия (что вырезаем — фон/изображение).
5. Простой набор параметров:
   - диаметр фрезы (мм),
   - глубина прохода (мм),
   - глубина за проход (мм),
   - скорость подачи (мм/мин),
   - обороты шпинделя (S),
   - safe Z, скорость перемещения.
6. Стратегия «растровая гравировка построчно» (zigzag): для каждой строки растра — если пиксель чёрный, фреза опускается, иначе поднимается. Простейший и надёжный путь для MVP.
7. Постпроцессор G-code для GRBL: G21, G90, G0/G1, M3/M5.
8. Сохранение `.gcode` в файл.
9. 2D-предпросмотр траектории на QGraphicsScene.

**Пример параметров MVP:**
```yaml
material:
  name: "Дерево (липа)"
  thickness_mm: 6
tool:
  type: "flat"
  diameter_mm: 1.0
job:
  depth_mm: 0.8
  step_down_mm: 0.4
  feed_xy: 600
  feed_z: 200
  spindle_rpm: 10000
  safe_z_mm: 5
```

**Пример выходного G-code (фрагмент):**
```gcode
G21         ; mm
G90         ; absolute
G17         ; XY plane
M3 S10000   ; spindle on
G0 Z5.000
G0 X10.000 Y10.000
G1 Z-0.400 F200
G1 X25.500 Y10.000 F600
G0 Z5.000
...
M5
G0 X0 Y0
M30
```

### Этап 2. Векторный ввод и контурная обработка

**Задачи:**
1. Импорт SVG: парсинг через `svgelements`, флэттенинг Безье в полилинии с заданной точностью.
2. Импорт DXF через `ezdxf` (LINE, ARC, LWPOLYLINE, SPLINE).
3. Учёт единиц измерения (mm/inch, преобразование из user units SVG).
4. Стратегия «по контуру» (Profile / Follow path).
5. Компенсация диаметра фрезы: внутренний/внешний оффсет через `pyclipper`.
6. Несколько проходов на глубину со step-down.
7. Lead-in / lead-out (касательный заход).
8. Оптимизация порядка контуров (минимизация холостых перемещений — простая жадная эвристика «ближайший сосед»).

**Пример работы оффсета:**
```
Исходный контур (буква O) → Pyclipper offset(-r) → внутренний контур → проход фрезой
```

### Этап 3. Стратегии для клише (главная фишка)

Клише для тиснения кожи — это, по сути, рельеф: фон выбран, изображение возвышается (или наоборот). Нужны специализированные стратегии.

**Задачи:**
1. **Pocketing (выборка кармана):** залить область вокруг изображения проходом фрезы. Реализация: последовательные оффсеты вглубь области (`pyclipper`), либо zigzag-заливка.
2. **V-carve (для острых углов и тонких деталей):** для V-фрезы — глубина пропорциональна локальной «толщине» области (рассчитывается через медиальную ось / distance transform). Это позволяет получать чёткие острые элементы.
3. **Рельеф из grayscale-изображения:** яркость пикселя → высота. Стратегия построчного 3D-сканирования (для финишного прохода).
4. **Чёрно-белое клише (типичный кейс для тиснения):**
   - чёрные области → возвышенности (то, что будет давить на кожу),
   - белые области → углубления (выбираются фрезой).
5. **Многоинструментная обработка:** черновик плоской фрезой + чистовик V-фрезой/гравёром в одном задании (либо два отдельных G-code файла).
6. **Инверсия для тиснения:** клише — это зеркальное отображение того, что будет на коже. Нужна кнопка «зеркалить по X» и/или «инвертировать рельеф».

**Пример V-carve:**
```
Для каждого пикселя внутри области:
    d = distance_transform(маска)[x, y]
    depth = min(d / tan(angle/2), max_depth)
    z(x, y) = -depth
```

### Этап 4. Профили материалов и фрез

**Задачи:**
1. Хранилище профилей в JSON/YAML в `~/.config/leathercam/` (Linux) и `%APPDATA%\leathercam\` (Windows).
2. Предзаполненные пресеты:
   - **Материалы:** липа, берёзовая фанера, ЛДФ, делрин, латунь (мягкая), алюминий, текстолит, эбонит.
   - **Фрезы:** 1/8" flat 1мм, 2мм, 3.175мм; V-bit 30°/60°/90°; гравёр 0.1мм; шаровая 1мм, 2мм.
3. Для каждой пары (материал × фреза) — рекомендации feed/speed/step.
4. UI редактора профилей: создание, копирование, удаление.
5. Импорт/экспорт пресетов (для обмена с другими пользователями).

**Пример профиля материала (YAML):**
```yaml
id: linden_wood
name: "Липа"
density_kg_m3: 530
recommended:
  - tool: vbit_30
    feed_xy: 800
    feed_z: 250
    step_down_mm: 0.3
    spindle_rpm: 12000
  - tool: flat_1mm
    feed_xy: 500
    feed_z: 150
    step_down_mm: 0.2
    spindle_rpm: 10000
```

**Пример профиля фрезы:**
```yaml
id: vbit_30
name: "V-bit 30°"
type: vbit
angle_deg: 30
tip_diameter_mm: 0.1
shank_diameter_mm: 3.175
flute_length_mm: 12
```

### Этап 5. Предпросмотр и симуляция

**Задачи:**
1. 2D-предпросмотр: отрисовка всех G0/G1 на QGraphicsScene с разными цветами (rapid/feed, разные Z).
2. Слайдер по строкам G-code → видно, как фреза «идёт» по детали.
3. Оценка времени: сумма (длина / feedrate) + время на rapid + время на смену Z.
4. Псевдо-3D: heightmap-симуляция — виртуальная заготовка как 2D-массив высот, фреза «вычитает» из неё свой профиль. Отображение через matplotlib/Qt3D/PyOpenGL.
5. Проверка границ: предупреждение, если траектория выходит за рабочее поле CNC 3018 (300×180×45 мм).
6. Подсветка коллизий (например, фреза в чёрной зоне на полной глубине).

**Задачи:**

2. Подключение к GRBL: чтение `$$`, статуса, версии.
3. Стриминг G-code построчно с учётом GRBL flow control (character-counting protocol).

6. Отображение текущей строки и прогресса.
7. **Важно:** этот модуль опционален — большинство пользователей шлют G-code через UGS/Candle. Делать его в самом конце.

## 9. Тестирование

1. **Unit-тесты** (pytest):
   - постпроцессор: проверка структуры G-code на эталонных примерах,
   - оффсет: известные геометрические случаи,
   - V-carve: depth от distance transform,
   - парсинг SVG/DXF: тестовые файлы.
2. **Snapshot-тесты G-code:** входной файл + параметры → ожидаемый G-code (с допуском по числам).
3. **Интеграционные тесты:** генерация G-code для набора эталонных задач, прогон через симулятор GRBL (например, `gcode-simulator` или собственный).
4. **Ручное тестирование:** реальные распилы на липе/фанере, фотографии результатов в репозиторий.
