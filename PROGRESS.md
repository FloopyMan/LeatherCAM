## Завершённые этапы

### Этап 1 — MVP (2026-05-28)

Полный конвейер `Image → Raster → Moves → G-code` работает end-to-end.
GUI на PySide6 с панелью параметров, загрузкой PNG/JPG, 2D-предпросмотром
и сохранением `.gcode`. 52 unit-теста, ruff чисто, GUI стартует под
offscreen Qt. Проверено на синтетическом PNG: 985 строк G-code,
многопроходная обработка по step-down.

### Этап 2 — векторный ввод и контурная обработка (2026-05-28)

Добавлены SVG/DXF импортёры (curve flattening, $INSUNITS), стратегия
Profile с pyclipper-offset (on/inside/outside), JobParameters расширен
до ProfileJobParameters, UI получил переключатель стратегии и пункт
"Открыть вектор". 99 unit-тестов. End-to-end проверка на SVG со
скруглённым прямоугольником и кругом → 1211 строк валидного G-code
с двумя step-down проходами.

Lead-in/lead-out и оптимизация порядка контуров отложены до Этапа 3
(стратегии клише) — добавим, если станут нужны на реальной фрезеровке.

### Этап 3 — стратегии для клише (2026-05-28)

Главная фишка проекта для тиснения кожи:

- **Pocket** (`leathercam/cam/pocket.py`): каскадные оффсеты с pyclipper,
  чистая выборка фона для замкнутых контуров.
- **V-carve** (`leathercam/cam/vcarve.py`): distance transform (scipy) +
  level sets (cv2.findContours). Угол V-фрезы и max_depth задаются;
  глубина квантуется по step_down.
- **Mirror toggle**: `leathercam/vector/transform.py::mirror_x` для
  векторов, `PIL.ImageOps.mirror` для растра. Сохраняет bbox.
- **Пресеты в меню**: «Клише для тиснения» (растровая + invert + mirror)
  и «Гравировка по линии» (растровая, без преобразований).

UI получил две новые стратегии в combo (Pocket, V-carve), группу
«V-carve» (угол + max depth), чекбокс «Зеркалить по X», меню «Пресеты».
130 unit-тестов. End-to-end: 300×150 PNG → клише 1465 строк, V-carve
882 строки с 5 проходами, Pocket 135 строк.

Heightmap → рельеф и многоинструментная обработка отложены: для клише
кожи Pocket+V-carve покрывают почти все кейсы; вернёмся, если станут
нужны.

Следующая остановка — Этап 4 (профили материалов и фрез).
