## Завершённые этапы

### Этап 1 — MVP (2026-05-28)

Полный конвейер `Image → Raster → Moves → G-code` работает end-to-end.
GUI на PySide6 с панелью параметров, загрузкой PNG/JPG, 2D-предпросмотром
и сохранением `.gcode`. 52 unit-теста, ruff чисто, GUI стартует под
offscreen Qt. Проверено на синтетическом PNG: 985 строк G-code,
многопроходная обработка по step-down.

### Этап 2 — векторный ввод и контурная обработка (2026-05-28)

Добавлены SVG/DXF импортёры (curve flattening, $INSUNITS), стратегия
Profile с pyclipper-offset (on/inside/outside), JobParameters расширен
до ProfileJobParameters, UI получил переключатель стратегии и пункт
"Открыть вектор". 99 unit-тестов. End-to-end проверка на SVG со
скруглённым прямоугольником и кругом → 1211 строк валидного G-code
с двумя step-down проходами.

Lead-in/lead-out и оптимизация порядка контуров отложены до Этапа 3
(стратегии клише) — добавим, если станут нужны на реальной фрезеровке.

### Этап 3 — стратегии для клише (2026-05-28)

Главная фишка проекта для тиснения кожи:

- **Pocket** (`leathercam/cam/pocket.py`): каскадные оффсеты с pyclipper,
  чистая выборка фона для замкнутых контуров.
- **V-carve** (`leathercam/cam/vcarve.py`): distance transform (scipy) +
  level sets (cv2.findContours). Угол V-фрезы и max_depth задаются;
  глубина квантуется по step_down.
- **Mirror toggle**: `leathercam/vector/transform.py::mirror_x` для
  векторов, `PIL.ImageOps.mirror` для растра. Сохраняет bbox.
- **Пресеты в меню**: «Клише для тиснения» (растровая + invert + mirror)
  и «Гравировка по линии» (растровая, без преобразований).

UI получил две новые стратегии в combo (Pocket, V-carve), группу
«V-carve» (угол + max depth), чекбокс «Зеркалить по X», меню «Пресеты».
130 unit-тестов. End-to-end: 300×150 PNG → клише 1465 строк, V-carve
882 строки с 5 проходами, Pocket 135 строк.

Heightmap → рельеф и многоинструментная обработка отложены: для клише
кожи Pocket+V-carve покрывают почти все кейсы; вернёмся, если станут
нужны.

### Багфикс — pocket для букв с дырками (2026-05-28)

Сообщил пользователь: для Б/О/Р/А/Д внутренний контур обрабатывался как
самостоятельный карман и выбирал то, что должно оставаться.

`leathercam/vector/grouping.py::group_with_holes` группирует замкнутые
полилинии по чётности глубины вложенности (через `shapely.contains` —
для целых полигонов, не representative_point); pocket теперь передаёт
в pyclipper outer (CCW) + дырки (CW) одним вызовом ET_CLOSEDPOLYGON.
Проверено на синтетическом кольце и реальном SVG с буквой "О": 0
точек реза попадает в центр. (commit: 5bdbebe)

### Этап 4 — профили материалов и фрез (2026-05-28)

`leathercam/profiles/` с моделями `Tool`/`Material`/`Recommendation`,
JSON-store через platformdirs (`~/.config/leathercam/`), 9 фрез по
умолчанию (flat 1/2/3.175 мм, V-bit 30/60/90°, шаровая, гравёр) и
6 материалов (липа, фанера, МДФ, делрин, латунь, алюминий) с
рекомендованными feed/speed/step для каждой пары.

В UI группа "Профиль материала и фрезы" с двумя combobox и кнопкой
"Применить рекомендованные параметры" — она подставляет feed_xy,
feed_z, spindle_rpm, step_down и диаметр (или угол для V-bit) из
выбранной пары. 157 unit-тестов.

Полноценный UI-редактор профилей и импорт/экспорт отложены — пока
хватает прямого редактирования JSON.

### Багфикс/доработки по фидбеку — Pocket "background" + workpiece + новые фрезы (2026-05-28)

По запросу пользователя:

- **Pocket "background" mode**: для клише на кожу нужно фрезеровать
  фон, а не рисунок. Engine добавляет workpiece-прямоугольник как
  outer глубины 0; группировка отдаёт инпуты как дырки → toolpath
  обходит дизайн, выбирая всё вокруг. Внутренние дырки (центр "О")
  тоже выбираются как островки фона. UI: combo "Режим (Pocket)" +
  поля «Размер клише, ширина/высота». Превью рисует контур заготовки.
- **Merge defaults on load**: новые фрезы/материалы в релизе теперь
  автоматически добавляются в `~/.config/leathercam/{tools,materials}.json`
  (по id), не теряя пользовательских правок. Опция `merge_defaults=False`
  для round-trip-тестов.
- **Новые фрезы**: tapered_ball_R0.25 / R0.1 / R0.5 (конические сфери­ческие
  TiAlN с углом A).
- **Новые материалы**: pmma_10mm (оргстекло), aluminium_amg3m_8mm (АМГ3М),
  aluminium_d16_14mm (Д16). С рекомендованными feed/speed/step для
  пар «материал × коническая фреза».

### Этап 5 — предпросмотр и симуляция (2026-05-28)

`leathercam/cam/metrics.py`: `BoundingBox`, `bounding_box(moves)`,
`estimate_duration_minutes(moves, feed_xy, feed_z, rapid_feed)`,
`exceeds_bounds(bbox, machine=CNC_3018_BOUNDS_MM, safe_z)`. Время
считается тем же правилом, что и feed-выбор в постпроцессоре.

UI: QSlider под превью «прокручивает» траекторию (рендерит первые N
moves); статус-строка показывает оценку времени работы и bbox; красная
строка под ней — предупреждения если путь выходит за рабочее поле
300×180×45 мм или Z поднимается выше safe Z. 177 unit-тестов.

Псевдо-3D heightmap и подсветка коллизий отложены — текущая визуализация
достаточно информативна.

### Этап 6 — отправка G-code на станок (2026-05-28)

`leathercam/grbl/transport.py`: `GrblTransport` с character-counting
flow control (128-байтный RX-буфер GRBL отслеживается в deque
inflight-строк). API: connect / disconnect / send_command / stream /
pause / resume / soft_reset / request_status / list_serial_ports.
`SerialLike` Protocol развязывает транспорт от pyserial (тесты
подсовывают `FakeSerial`).

UI: меню «Файл → Отправить на станок…» (Ctrl+Shift+S) открывает
`MachineDialog` — выбор порта/скорости, ручная команда, jog-сетка
X/Y/Z± с шагом и подачей, Home/$H, Zero/G92, прогресс-бар стриминга
с кнопками Старт/Пауза/Продолжить/Стоп. Стриминг идёт в `QThread`
через `StreamWorker`.

13 unit-тестов транспорта (FakeSerial). 190 всего.

### Этап 7 — полировка UX (2026-05-28)

- **Drag&drop**: PNG/JPG/BMP/SVG/DXF на главное окно — `_open_path`
  диспатчит по расширению.
- **Recent files**: подменю «Файл → Недавние файлы» через `QSettings`
  (до 5 записей). Каждое успешное открытие добавляет путь, есть кнопка
  «Очистить список».
- **Тёмная/светлая/системная тема**: меню «Вид → Тема», сохраняется в
  `QSettings`, применяется через `QApplication.setStyleSheet`.
- **Логирование**: rotating file handler в `~/.local/state/leathercam/log/`
  (или `%LOCALAPPDATA%\leathercam\Logs\` на Windows), 1 МБ × 5 ротаций.
  Дублируется в консоль.

Локализация и расширенные подсказки полей отложены — UI пока на русском.

4 новых UI-теста + рефакторинг загрузки файлов в общий `_open_path`.
